# 04. Plan de desarrollo

## Fase 1. Auditoría y preparación
- ordenar repositorio
- inspeccionar Excel real
- generar normalización borrador
- documentar hallazgos, riesgos y preguntas abiertas

## Fase 2. Definición funcional
- validar hallazgos con cliente
- decidir hoja base y reglas de corte
- cerrar requerimientos MVP
- confirmar reglas de producto, inventario y venta rápida

## Fase 3. Diseño de datos y arquitectura
- definir modelo de datos final
- definir stack backend/frontend
- diseñar flujo de importación
- diseñar auditoría y trazabilidad mínima

## Fase 4. Implementación inicial
- catálogo de productos
- inventario actual
- movimientos básicos
- venta rápida
- reportes iniciales

## Fase 5. Endurecimiento operativo
- pruebas de datos reales
- depuración de catálogos
- importaciones controladas
- control de usuarios y turnos
- estabilización de reportes

## Criterios para avanzar a Fase 2
- validar interpretación de las hojas del Excel
- validar columnas y semántica de movimientos
- acordar hoja recomendada como inventario inicial
- resolver principales anomalías de datos críticas
